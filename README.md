# Свадебный сайт-приглашение — инструкция по запуску

## Что уже сделано
- `index.html` — сам сайт (обложка с "печатью", открытие → музыка, программа дня,
  дресс-код, локация, форма RSVP с отправкой ответа в сообщения ВКонтакте).
- `.github/workflows/deploy.yml` — при каждом пуше в `main` автоматически
  подставляет токен доступа из GitHub Secrets и публикует сайт на GitHub Pages.
- `assets/` — сюда нужно положить свою музыку и фотографии (см. ниже).

Отправка сделана через ВК (а не Telegram) специально: у части гостей без VPN
`api.telegram.org` может быть заблокирован на уровне провайдера, и форма
зависала на этапе отправки. `vk.com` — российская платформа, этой проблемы нет.

Токен доступа **нигде не хранится в открытом виде в этом репозитории** — в коде
стоит заглушка `__VK_TOKEN__`, которую workflow подставляет во время
сборки, беря значение из зашифрованных GitHub Secrets.

> Важно понимать границу этой защиты: если репозиторий **публичный**, то и
> собранная страница на GitHub Pages тоже публична — а значит токен всё равно
> будет виден в исходном коде готовой страницы (через "Просмотр кода страницы"
> или вкладку Network в браузере), просто не в самом репозитории на GitHub.
> Для свадебного сайта с небольшим числом гостей это адекватный компромисс.
> Если захотите спрятать токен полностью даже от заинтересованного гостя —
> напишите мне, сделаем через отдельную серверную функцию (Netlify/Vercel
> Function) — это отдельный, чуть более сложный вариант.

## Шаг 1. Получите токен доступа ВКонтакте

1. Создайте сообщество ВКонтакте (или используйте существующее).
2. **Управление сообществом → Работа с API → Ключи доступа** → создайте ключ
   с правом `messages` (сообщения).
3. Сохраните токен вида `vk1.a....`.
4. Для каждого получателя: он должен один раз написать сообществу любое
   сообщение (например «привет»), чтобы у сообщества появилось право писать
   ему в ответ.
5. Узнайте `user_id`/`peer_id` каждого получателя (например, через
   `https://api.vk.com/method/messages.getConversations?access_token=<ТОКЕН>&v=5.199`
   и поиск нужного диалога в ответе).

## Шаг 2. Создайте репозиторий на GitHub

1. На github.com → New repository → назовите, например, `wedding`.
2. Публичный или приватный — на ваш выбор (см. предупреждение выше).
3. Загрузите туда все файлы из этой папки (через "Add file → Upload files"
   в браузере, либо через `git push`, если умеете пользоваться git).

## Шаг 3. Добавьте секреты

В репозитории: **Settings → Secrets and variables → Actions → New repository
secret**. Добавьте секреты:

| Имя секрета | Значение |
|---|---|
| `VK_ACCESS_TOKEN` | токен доступа из Шага 1 |
| `VK_USER_ID_1` | user_id/peer_id первого получателя |
| `VK_USER_ID_2` | (необязательно) user_id/peer_id второго получателя |

Если второй получатель не нужен — просто не добавляйте секрет `VK_USER_ID_2`,
сайт продолжит работать с одним получателем.

## Шаг 4. Включите GitHub Pages

**Settings → Pages → Source** → выберите **GitHub Actions** (не "Deploy from
a branch").

## Шаг 5. Запустите публикацию

Сделайте любой коммит в ветку `main` (или зайдите во вкладку **Actions** и
нажмите "Run workflow" вручную). Через 1-2 минуты сайт появится по адресу
вида `https://ваш-логин.github.io/wedding/`.

## Как отредактировать содержимое

Все тексты (имена, дата, место, программа дня, дресс-код, адрес) находятся
прямо в `index.html` — они помечены понятными блоками (`<section id="...">`).
Просто откройте файл и замените текст на свой, ничего в разметке ломать не
нужно.

## Музыка

Положите файл в `assets/` и укажите его имя в `index.html`, строка
`<audio id="bgAudio" src="assets/..." ...>`.

Музыка начинает играть только по нажатию на "печать" на заглавном экране —
это не баг, а требование браузеров: со звуком автоматически ничего играть
не разрешено, только по действию пользователя.

**Про авторские права**: если это трек популярного исполнителя — не
выкладывайте его в **публичный** репозиторий (это будет незаконным
распространением). Либо сделайте репозиторий приватным, либо возьмите
royalty-free мелодию (например, с Pixabay Music, Free Music Archive) или
трек, на который у вас есть права использования.

## Фотографии

Кладите в `assets/`, ссылайтесь из `index.html` как `assets/имя-файла.jpg`.
Если фото личные — то же самое: думайте о приватном репозитории, если не
хотите, чтобы они были доступны кому угодно по прямой ссылке на файл в GitHub.

## Подтверждение RSVP и подарочек-анимация

При успешной отправке формы на экране появляется всплеск конфетти
(библиотека `canvas-confetti`, подключается с CDN, ничего дополнительно
настраивать не нужно).
